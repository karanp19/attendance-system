\section{Introduction}
One-shot learning is a machine learning model that enables the model to learn and make predictions based on a small number of examples. It is beneficial in situations where it is hard or impractical to collect a lot of training data. One-shot learning has been used to do a lot of different things, like classifying images, translating languages, and keeping track of who was there and when. A one-shot learning attendance system is usually made up of a camera or other device that takes pictures or video of people entering or leaving a building or other restricted area. The system also includes a machine-learning model trained to recognise the faces of individuals authorised to access the area.For the system administrator to train the model, they must first take a few pictures of each person who is allowed to enter the area. These images are used to create a "reference" dataset that the model can use to learn the characteristics of each person's face.\cite{1}\\

Once the model has been trained on the reference dataset, it can be used to recognise the faces of individuals as they enter or exit the restricted area. When the image capture device takes a picture of a person's face, the model compares the picture to the reference dataset to determine if the person is allowed to enter the area. The model allows the person to enter or exit the area if the person is authorised. If the person is not allowed in, the model can do a number of things, such as deny access, send an alert to security staff, or take a picture of the person so they can be identified later. One-shot learning has several advantages for attendance tracking systems. First, it allows the system to be trained on a few examples, reducing the amount of time and resources required to set up the system. Second, it lets the system recognise people even if their looks have changed slightly since the reference images were taken. For example, their hairstyle or facial hair might have changed. Lastly, one-shot learning can be used to recognise a lot of people with a small amount of training data. This makes it a good choice for large organisations with a lot of employees or other people who need to be recognised. There are also some challenges associated with using one-shot learning for attendance tracking. One challenge is that the system will only be accurate if the reference images are high quality or accurately represent the individual's appearance. Also, the system will have a more challenging time recognising people who are wearing masks or whose appearance has changed a lot since the reference images were taken. Finally, one-shot systems will be more vulnerable to adversarial attacks, where an individual attempt to deceive the system by presenting it with a forged or altered image.\cite{2}\\

In summary, an attendance system using one-shot learning is a type of machine learning system that can recognise the faces of individuals based on a small number of reference images. It's helpful in keeping track of attendance when it's hard or impractical to collect a lot of training data, and it can recognise people even if they've changed a little since the reference images were taken. However, it can only be accurate if the reference images are of high quality or accurately represent the individual's appearance. It can be less effective at recognising individuals wearing disguises or significantly altering their appearance. \subsection{Motivation}

One thing that can be particularly time-consuming is taking attendance in large classes. One way to streamline this process is by using one shot learning to create an attendance system. One shot learning is defined as a type of machine learning where a model is extensively trained on a surprisingly small number of examples, and then is able to make predictions about new, unseen data. This can be especially useful in the context of attendance systems because it allows the model to quickly and accurately identify students based on a single image of their face.\\
There are several benefits to using a one shot learning-based attendance system. First and foremost, it saves time. Instead of having to manually take attendance, the system can do it automatically, freeing up the instructor to focus on other tasks. This can be especially useful in large classes where it might take several minutes to take attendance the traditional way. Another benefit of using a one shot learning-based attendance system is that it can be more accurate than a manual system. When attendance is taken manually, there is always the risk of error, whether it's a student forgetting to sign in or the instructor misremembering who was present. With a one shot learning-based system, the model is trained in such a way that it is able to accurately identify each student, reducing the risk of error. In addition to saving time and increasing accuracy, using a one shot learning-based attendance system can also be more convenient for students. Instead of having to remember to sign in or raise their hand, they can simply walk into class and have their attendance recorded automatically. This can be especially useful for students who are running late or who are otherwise unable to physically sign in.

\section{Theoretical background and Literature Survey}

Attendance systems are used to track and record the attendance of
students or employees in a variety of settings, including schools,
colleges, and workplaces. One-shot learning is an approach to machine
learning that lets a model learn from a small number of examples and
make predictions based on what it has learned. This makes it a promising
method for attendance systems. One-shot learning has been applied to attendance systems in a number of ways. For example, one-shot learning algorithms have been used to
recognize the faces of students or employees and record their attendance
based on this recognition. This type of system is known as a "facial
recognition attendance system." Other one-shot learning approaches have
been used to track attendance by recognizing patterns in other types of
data, such as handwriting or signatures.A number of one-time learning algorithms, such as Siamese neural networks, prototypical networks, and metric-based methods, have been
used in attendance systems. These algorithms have been successful at
many things, like checking a face and figuring out what it is. But
one-shot learning algorithms still need help, like how they need a lot
of data to train and how hard it is for them to learn about new classes.

Siamese neural networks, prototypical networks, and metric-based methods
are all techniques used in machine learning for various tasks, including
image classification, object recognition, and natural language
processing.Siamese neural networks are a type of neural network architecture
consisting of two identical neural networks, each with its own weights
and biases. These networks are trained to process input data in the same
way, and their outputs are compared to one another. Siamese neural
networks are often used for tasks that involve comparing two input
samples, such as determining whether two images contain the same object
or whether two sentences convey the same meaning.\cite{12}

Prototypical networks, on the other hand, are a type of neural network
architecture that is specifically designed for few-shot learning, a
machine learning task where the goal is to classify a small number of
examples from each class. In prototypical networks, the network is
trained to learn a prototype or representative example for each class,
and new examples are classified based on their similarity to these
prototypes.Metric-based methods, also known as distance-based methods, are a type
of machine learning approach that involves measuring the distance or
similarity between examples in order to classify them. These methods are
often used in combination with other machine learning techniques, such
as k-nearest neighbors (k-NN) or support vector machines (SVMs), to
classify data based on how similar it is to previously seen examples.\cite{11}

In summary, Siamese neural networks are used for comparing two input
samples, prototypical networks are used for few-shot learning, and
metric-based methods are used for classifying data based on the
similarity between examples. In addition to one-shot learning algorithms, attendance systems may also use other technologies, such as RFID (Radio-Frequency Identification) or
QR codes, to track attendance. These systems might be better in some
situations, like when there are a lot of students or employees or when
facial recognition can't be used because of privacy concerns. Overall, one-shot learning has shown promise as an approach for attendance systems, enabling the development of systems that can
accurately track and record attendance with a small number of examples.
But more research needs to be done to find out how one-time learning in
attendance systems work and what problems and limits they have.\cite{43}

\subsubsection{Dimensionality Reduction}
Dimensionality reduction is the process of minimizing the number of dimensions or features in a dataset while maintaining as much of the original information as possible. It is a widely used technique in machine learning and data analytics, serving to decrease data complexity, enhance model interpretability, and boost the performance of machine learning algorithms. Dimensionality reduction techniques can be broadly divided into two categories: feature selection and feature extraction.

Feature selection focuses on choosing a subset of the original features in the dataset based on their relevance or significance to the task at hand. There are various methods for feature selection, such as:

Filter methods: These techniques employ statistical measures to assess the relevance of each feature, selecting the most pertinent features based on a predefined threshold. T-test and chi-squared test are examples of filter methods.

Wrapper methods: These techniques utilize a machine learning algorithm to evaluate the performance of different feature subsets and select the subset yielding the best performance. Forward selection and backward elimination are examples of wrapper methods.

Embedded methods: These techniques incorporate the selection of relevant features during the optimization of a machine learning algorithm as part of the model training process. Lasso and ridge regression are examples of embedded methods.\\
Feature extraction, on the other hand, involves generating a new set of features from the original dataset using projection or transformation techniques. Several methods for feature extraction exist, including:

Principal Component Analysis (PCA): This method projects data onto a lower-dimensional space using linear combinations of the original features. The resulting components are ranked by their importance, with the most important components retained for further analysis.

Multidimensional Scaling (MDS): This method projects data onto a lower-dimensional space while preserving the pairwise distances between the points.

Isomap: This method employs a graph-based approach to project data onto a lower-dimensional space while maintaining the global structure of the data.

t-SNE: This method utilizes a probabilistic model to project data onto a lower-dimensional space, preserving the local structure of the data.\cite{41}\cite{40}\\
In addition to these techniques, hybrid approaches that combine feature selection and feature extraction methods also exist. Dimensionality reduction can be an effective tool for decreasing data complexity and enhancing the performance of machine learning algorithms. However, it is crucial to thoughtfully consider the appropriate technique for a specific dataset and task, as different techniques may have unique strengths and limitations.

It is also important to take into account the trade-off between reducing dimensionality and the potential loss of information, as decreasing the number of dimensions could result in the loss of vital information in the data. Several factors should be considered when selecting a dimensionality reduction technique, such as the nature of the data, the complexity of the task, and the available computational resources.

For instance, if the data is highly structured and the task is relatively simple, a feature selection method might be adequate. Conversely, if the data is highly complex and the task is more challenging, a feature extraction method might be more suitable. Computational resources should also be considered, as some dimensionality reduction techniques can be computationally demanding. For example, PCA and MDS are generally faster than t-SNE, but t-SNE may be more effective in preserving the local structure of the data.

Additionally, it is crucial to consider the interpretability of the resulting dimensions. Some dimensionality reduction techniques, like PCA, produce dimensions that are more interpretable than others, such as t-SNE. This can be important for tasks like data visualization or feature analysis.   In summary, dimensionality reduction is a powerful tool for simplifying data and enhancing the performance of machine learning algorithms. It is crucial to carefully consider the most suitable technique for a specific dataset and task and to weigh the trade-off between reducing dimensionality and the potential loss of information.

By taking into account factors such as the nature of the data, the complexity of the task, the available computational resources, and the interpretability of the resulting dimensions, practitioners can make informed decisions about which dimensionality reduction technique to employ. Ultimately, this can lead to more effective and efficient data analysis, improved model interpretability, and better-performing machine learning algorithms.\cite{42}
\subsubsection{CNN}
Convolutional neural networks (CNNs) are a specialized kind of artificial neural network designed for processing grid-like data structures, such as images. They have become integral in numerous applications, including image classification, object recognition, and image generation. CNNs consist of several layers of interconnected neurons that process and manipulate input data through multiple mathematical operations.

The initial layer of a CNN is the input layer, which takes in raw data. Following this are hidden layers that perform various transformations on the data to extract meaningful features. The final layer, known as the output layer, generates the ultimate prediction or classification. A distinctive aspect of CNNs is the incorporation of convolutional layers, which execute a convolution operation on the input data. The convolution process applies a compact, fixed-size filter to the input data and slides it across the data to extract features. This enables CNNs to capture local features critical for tasks like image classification and object detection.

Additionally, CNNs utilize pooling layers that downsample input data by selecting the maximum or average value within a small data region. Pooling layers aid in reducing input data size and boosting the CNN's computational efficiency. CNNs have demonstrated exceptional performance in various tasks, such as image classification, object detection, and image generation. They are widely employed in industry and research and have become a fundamental tool in computer vision.

However, CNNs have some limitations. One notable drawback is their dependency on extensive labeled data for training, which can be costly and time-consuming to gather.\cite{44}\\

Furthermore, CNNs' performance can be influenced by hyperparameter choices, such as the learning rate and the number of layers. Lastly, CNNs can be susceptible to overfitting, which occurs when the model becomes excessively specialized to the training data, resulting in poor performance on new data.

In recent years, several advancements in CNNs have enhanced their performance and broadened their potential applications. One such advancement is the implementation of deep CNNs, which are CNNs with a high number of layers. Deep CNNs have achieved outstanding performance in tasks like image classification, object detection, and image generation. Another significant development is the application of transfer learning, where the weights and biases of a pre-trained CNN are used as a starting point for training a new CNN on a different task. This can substantially reduce the labeled data needed for training and improve the new CNN's performance.

\subsubsection{SHA-256}

The Secure Hash Algorithm 256-bit (SHA-256) is a cryptographic hash function that produces a fixed-length digital signature from a given data input. As one of the multiple cryptographic hash functions outlined by the National Institute of Standards and Technology (NIST) in the Federal Information Processing Standard (FIPS) 180-4, SHA-256 is frequently employed to confirm the authenticity of data in contexts such as digital signatures and software updates, as well as to generate secure passwords and keys. The National Security Agency (NSA) introduced SHA-256 in 2001 as a component of the Secure Hash Standard (SHS), which specifies a range of cryptographic hash functions for use in federal government applications. The initial SHA hash function emerged in 1993 but was later discovered to have vulnerabilities to specific attacks, prompting the creation of more robust hash functions like SHA-256.\cite{avast_md5_hashing}\\

Belonging to the SHA-2 family of hash functions, SHA-256 is accompanied by other variants such as SHA-224, SHA-384, and SHA-512. Regarded as more secure and dependable than its forerunner, SHA-1, SHA-256 has replaced the earlier version, which has since been deemed insecure due to multiple discovered vulnerabilities. SHA-256 operates as a cryptographic hash function that accepts a variable-length input and yields a fixed-size output, often referred to as a hash or digest. The resulting output is a unique 256-bit (32-byte) value corresponding to the input data. Identical inputs consistently generate the same outputs, but even minor alterations to the input yield drastically different outputs. This characteristic, called the "avalanche effect," renders it challenging to foresee the function's output based solely on its input.\cite{39}\\

A crucial characteristic of SHA-256 is its resilience against collision attacks, which involve attempts to identify two distinct inputs that yield the same hash value. SHA-256 is structured to render it computationally impractical to discover two inputs with matching hash values, even after numerous trials. Additionally, the speed and efficiency of SHA-256 make it suitable for a wide array of applications. It can be implemented through software or hardware and is often employed alongside other cryptographic methods, such as digital signatures and message authentication codes (MACs).

SHA-256 sees use in various applications, encompassing digital signatures, software updates, and secure password storage. It also plays a role in the Bitcoin blockchain and other cryptocurrencies, where it safeguards transactions and validates the blockchain's integrity. As per a study by the Cryptographic Hash Function Database, SHA-256 ranks as the second most prevalent cryptographic hash function, holding a 26.5 percent market share. Moreover, it is the most popular variant within the SHA-2 family of hash functions, boasting a 99.9 percent market share.

In conclusion, SHA-256 is a widely adopted and dependable cryptographic hash function specifically designed to resist collision attacks while offering speed and efficiency. It is frequently utilized to ensure data integrity, generate secure passwords and keys, and protect transactions within the Bitcoin blockchain and other cryptocurrencies.\cite{sciencedirect_cryptographic_hash}

Introduced by Viola and Jones in 2001, this method uses a set of simple features to detect and track objects in images and videos. Since then, Haar Cascades have been widely adopted and have become a fundamental tool in computer vision.\cite{76}\cite{77}\\

Haar Cascades represent classifiers that identify objects by examining distinct sub-regions within an image or video. These classifiers are developed by training a machine learning model using positive and negative samples. Positive samples encompass images featuring the object of interest, while negative samples consist of images lacking the target object. Haar Cascades rely on basic features, such as differences in brightness between neighboring pixels, as well as more intricate features related to texture and edge detection.

During the training process, the machine learning model learns to distinguish between the two sample categories by adjusting its weights to minimize classification errors. Once the model has been trained, it can detect the object of interest in new images and videos. This is achieved by scanning the image using a sliding window and applying the classifier to each sub-region.\cite{78}\cite{79}\\

Haar Cascades have several advantages over other object detection methods. Firstly, they are computationally efficient and can detect objects in real-time, making them suitable for applications such as face detection and tracking. Secondly, they are robust to changes in illumination and can detect objects under different lighting conditions. Thirdly, they can be trained on a small number of samples, making them ideal for applications where large training sets are not available. Haar Cascades have been used in a wide range of applications, including face detection, object tracking, pedestrian detection, and gesture recognition. Face detection is one of the most popular applications of Haar Cascades, which has been widely used in security systems, social media, and virtual reality. Haar Cascades have also been used in object tracking, where they can be used to track the movement of objects in real-time, and in pedestrian detection, where they can be used to detect pedestrians in traffic and prevent accidents. Moreover, Haar Cascades have been used in gesture recognition, where they can be used to identify hand and body movements in real-time.\cite{80}\\

Over the years, several improvements have been made to the Haar Cascade algorithm to make it more efficient and accurate. One such improvement is the use of Adaboost, which is a boosting algorithm that improves the accuracy of the Haar Cascade classifier. Another improvement is the use of LBP (Local Binary Patterns) features, which are more robust to changes in lighting conditions and provide better accuracy compared to Haar features. Additionally, some researchers have explored the use of deep learning techniques, such as convolutional neural networks (CNNs), for object detection and tracking.\cite{81}\\

Haar cascades are a cascade of weak classifiers that gradually eliminate non-object regions of an image. A Haar cascade consists of multiple stages, with each stage consisting of multiple weak classifiers. Each weak classifier evaluates a single feature or set of features, which are typically rectangular regions of the image. At each stage, the classifier determines whether the image patch contains the object of interest or not. If the patch is rejected, it is discarded and the next stage is evaluated. If the patch passes all stages, it is considered a positive detection.\cite{83}\cite{84}\\


The quantity of stages and weak classifiers in Haar Cascade object detection is contingent upon the object's complexity and the desired level of accuracy. Generally, a larger number of stages and weak classifiers leads to greater accuracy but a slower detection process.

At the core of Haar Cascade object detection lies the utilization of Haar-like features, which are rectangular sections of the image exhibiting varying intensity levels. These features are calculated by subtracting the sum of pixel intensities in a rectangular region above the feature from the sum of pixel intensities in a rectangular region below the feature. The outcome is a singular value that represents the difference in intensity levels and serves as a feature for the classifier.

Haar features are computationally efficient, allowing for simultaneous computation of multiple features using integral images. An integral image is one in which each pixel denotes the sum of all pixels above and to its left. The implementation of integral images facilitates rapid computation of Haar features, thereby enhancing the efficiency of the detection process.\cite{85}\\

The training of Haar cascades involves selecting a set of positive and negative images and iteratively optimizing the weak classifiers to minimize the classification error. The optimization process involves selecting the best feature for each weak classifier and determining the optimal threshold for each feature.
One optimization technique for Haar cascades is AdaBoost, a machine learning algorithm that selects the most informative features and optimizes the weak classifiers. AdaBoost trains each weak classifier on a subset of the training data, with more weight given to misclassified examples. This results in a stronger classifier with higher accuracy.\cite{86}\cite{87}\\

Another method for optimization involves cascading, in which weak classifiers are organized hierarchically, with features becoming progressively more complex. This approach minimizes false positives and enhances overall detection accuracy.

In summary, Haar Cascades serve as a potent instrument for object detection within the realms of computer vision and machine learning. They offer computational efficiency, remain robust under varying lighting conditions, and can be effectively trained using a limited number of samples. Haar Cascades have been extensively applied in numerous use cases, such as face detection, object tracking, pedestrian detection, and gesture recognition. As the fields of computer vision and machine learning continue to evolve, Haar Cascades are expected to maintain their significance as a key tool for object detection and tracking.\cite{82}\cite{88}\cite{89}\cite{90}\\

One-shot learning represents a machine learning approach that enables the classification or recognition of new classes or categories using just a single or a few examples. This differs from conventional machine learning, which often demands a substantial volume of labeled data to learn and generalize to new classes. One-shot learning proves especially valuable in circumstances where acquiring large quantities of labeled data is difficult or impractical, such as when dealing with rare or unique classes or when the time or cost of labeling data is high.

The limited availability of training data makes one-shot learning a challenging task. To generalize to new classes based on minimal information, models must effectively extract and employ pertinent features and patterns from the data. This necessitates the creation of robust and adaptable models capable of handling noise and variability in the data, as well as quickly and accurately adapting to new classes. Various strategies for one-shot learning exist, including metric-based methods, model-based methods, and meta-learning methods.\cite{13}\\

Metric-based methods involve learning a distance or similarity metric between examples in order to compare and classify new examples. One example of a metric-based method is the use of matching networks, which use a learned metric to compare the input to a set of support examples and predict the label of the input based on the similarity between the input and the support examples. Another example is the use of prototypical networks, which use a learned metric to compute the distance between the input and the prototypes of each class, and then use the distances to predict the label of the input. Model-based methods involve learning a model or function that maps inputs to outputs, and then using the model to classify new examples. One example of a model-based method is the use of graph neural networks (GNNs) for one-shot learning. GNNs use a graph structure to encode the relationships between the input and the support examples, and then use the encoded relationships to predict the label of the input. Another example is the use of relational information in one-shot learning, where a model uses a graph neural network to encode the relationships between the input and the support examples, and then uses the encoded relationships to predict the label of the input.\cite{14}\\

Meta-learning, also known as learning to learn, centers on acquiring a model or a set of parameters that can be quickly adapted to new tasks with only a few examples. By leveraging a differentiable convex optimization algorithm, the model is capable of learning parameters that can adjust to new tasks even when data is limited. \subsection{Model Summary}

Figure 5 is a picture that shows how the embedding layer of our model is
put together. The embedding layer is a key part of our model because it
takes the input data (in this case, images) and turns it into a
numerical representation that the rest of the model can use. The picture shows that the input data goes through a extended and comprehensive series of
convolutional as well as pooling layers, which pull out specific features from the input
data and reduce its dimensionality. The results of these layers are then
sent to the embedding layer, which turns the extracted features into
numbers. This numerical representation, called an embedding, is a
simplified data version showing the most important patterns and
features.

In deep learning models, the embedding layer is often used to do things
like classify images, find objects, and process natural language. It
allows the model to learn a compact representation of the input data
that is more efficient to process and that captures the most important
features and patterns. In our model, the embedding layer is a key part
of how the model learns from a small number of examples and uses this
learning to make accurate predictions.

The image shown in Figure 6 summarizes the structure of our Siamese
network. A Siamese network is a type of neural network that consists of
two identical sub-networks that are trained to recognize similar
features in two input images. The output of a Siamese network is a
measure of the similarity between the two input images. The picture shows that the Siamese network has two branches with several
convolutional and pooling layers on each. The input images are passed
through these branches, and the output of the branches is fed into a
final layer, which compares the outputs and calculates a measure of the
similarity between the input images.\\

Siamese networks have been used for a wide range of tasks, such as
recognizing objects, checking faces, and processing natural language.
They are particularly useful in one-shot learning, as they are able to
learn and make predictions based on a small number of examples. In our
model, the Siamese network is a key part of how the model can find
similar parts in different images and figure out how similar they are.
This helps the model accurately classify the input images and make
predictions based on this classification.
